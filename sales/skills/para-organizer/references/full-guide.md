# Guide : La Méthode d'Organisation PARA

PARA est une structure universelle pour organiser tous types d'informations numériques.

## 1. Les 4 Piliers

1. **Projets** (1-Projects) : Travail actif avec une date de fin (ex: "Lancer le site web").
2. **Domaines** (2-Areas) : Responsabilités continues qui exigent un standard élevé (ex: "Finances", "Santé", "Marketing").
3. **Ressources** (3-Resources) : Sujets d'intérêt ou de référence (ex: "Idées de design", "Bibliothèque React", "Recettes").
4. **Archives** (4-Archives) : Tout ce qui est terminé ou inactif.

## 2. Flux de la Boîte de Réception (Inbox Review)

Tous les nouveaux fichiers arrivent dans `0-Inbox`. Posez-vous ces questions :

- Est-ce actionnable ? -> **Projet**.
- Est-ce une responsabilité ? -> **Domaine**.
- Est-ce utile plus tard ? -> **Ressource**.
- Sinon -> **Archive** ou **Supprimer**.

## 3. Convention de Nommage Pro

Un nom de fichier doit être compréhensible sans être ouvert.
Format : `[DATE]_[CODE]_[DESCRIPTION]`
_Exemple : `2025-01-20_FIN_facture-serveur.pdf`_

## 4. Journalisation et Sécurité

L'organisateur PARA maintient trois fichiers de suivi dans le dossier `_ORG/` :

- `_PLAN.md` : Liste des tâches d'organisation.
- `_LOG.md` : Actions effectuées par l'IA.
- `_MANIFEST.md` : Piste d'audit pour permettre un retour en arrière (Rollback).
